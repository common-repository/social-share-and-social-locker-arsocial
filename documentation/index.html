ARSocial is an exclusive WordPress plugin that is a combination of Social Share, Like / Follow / Subscribe, Social Locker, and Social Fan Counter.

ARSocial provides a major feature which is global settings to display social share, social like, social locker and social fan counter without placing shortcode at each page or posts

Installation and Activation

Steps For Installation And Activation

You can install plugin by either of these two methods.

New install via the WordPress Uploader

  1. Download the plugin from CodeCanyon
  2. Log in to your WordPress admin panel
  3. Go to Plugins -> Add New -> Upload
  4. Click on choose file and browse the download package from CodeCanyon. When the plugin uploads successfully, click on Activate Plugin.

After activating the plugin, you will be able to see ARSocial menu in admin left menu.

After activating the plugin, you need to activate your license for enablling support and automatic updates. A license key only be used for one installation of WordPress at a time.

Click on Licensing link to activate your license. You will see below section there on that page. To activate your license, you need to enter your Name, Email address and purchase code ( Unique code obtained while purchased this product ).

New install via FTP

  1. Download the plugin from CodeCanyon, and extract it. You should find a directory arsocialshare.
  2. In your WordPress installation, upload arsocialshare to wp-content/plugins/
  3. Activate the ARSocial plugin through the Plugins menu from WordPress admin panel.

General Settings

After installing and activating plugin, now its time to configure social networks before social buttons configuration. So, go to the General Settings menu from the admin interface. You have sections for Social Network Configuration, Email Configuration, URL Shortner, After Share Action and Front-end Font Settings.

Social Network Configuration

As you can see in below image, here is list of social networks where you have to fill required API information related to particular social network.

Facebook

To use Facebook API you should have Facebook Application ID which you can get using this link. Or you are also able to get more detail at our Facebook page at External API section.

You have another option to select Facebook Application Version.

Twitter

Twitter Application requires Consumer Key, Consumer Secret, Access Token and Token Secret here.

Now get your Application details from Twitter API. If you are not able to know how to get it then visit our Twitter page at External API section.

VKontakte

Fill VKontakte Application Id here. Create your new application from VKontakte API or you can also follow steps as given at our VKontakte page at External API section.

LinkedIn

If you are going to use LinkedIn then you have to provide API Key here.

If you are aware of steps to get Application Key then you can get it from LinkedIn API, otherwise you are able to get detail at our LinkedIn page at External API section.

Bitly

If you are going to use Bitly service as URL Shortner then you have to provide Access Token here.

You can get Access Token from Bitly API, please visit our Bitly page at External API section to get detail.

Dribbble

If you are going to use Dribbble then you have to provide Client ID and Client Secrethere and need to generate access token.

You can get Client ID and Client Secret from Dribbble API, please visit our Dribbble page at External API section to get detail.

Load JS & CSS in pages

Generally ARSocial loads JS and CSS files only when needed and only for specific type of social buttons which you are displaying at front-end.

But in rare case where you have placed share button using Shortcode Method and your theme is ajax based, then it might be possible that those additional css/js do not load, as in ajax theme header and footer do not reload while you navigate through pages.

Only in that case you may need to enable loading js/css explicitly.

You have four options here as you have seen in above image to include different type of js/css as per your need

Bitly API

In order to get a access token for your account please login to your account at https://bitly.com/.

Access Token

You are able to get access token for your account from https://bitly.com/a/oauth_apps where you have to login to your account first.

You may also follow the Basic Authentication Flow to get access token.

  • Register your application here at http://bitly.com/a/oauth_apps where your application will be assigned a client_id and a client_secret.
  • Redirect the user to https://bitly.com/oauth/authorize, using the client_id and redirect_uri parameters to pass your client ID and the page you would like to redirect to upon acquiring an access token.
  • Upon authorizing your application, the user is directed to the page specified in the redirect_uri parameter. bitly append a code parameter to this URI that contains a value that can be exchanged for an OAuth access token using the oauth/access_token endpoint.
  • Use the /oauth/access_token API endpoint to acquire an OAuth access token, passing the code value appended by bitly to the previous redirect and the same redirect_uri value that was used previously.
  • This API endpoint will return an OAuth access token, as well as the specified bitly user's login and API key, allowing your application to utilize the bitly API on that user's behalf.
